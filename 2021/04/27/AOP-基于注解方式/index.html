AOP 基于注解方式

AOP

Aspect Oriented Programming

利用AOP可以对业务逻辑的各个部分进行隔离，从而使得业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发的效率。

即在不通过修改源代码的情况下，添加新功能。

底层原理

动态代理

区分两种情况

  1. 有接口情况，使用 JDK 动态代理

    创建接口的实现类代理对象，增强类的方法

  2. 没有接口情况，使用 CGLIB 动态代理

    创建当前类子类的代理对象，增强类的方法

即，Spring 对接口类型使用 JDK 动态代理，对普通类使用 CGLIB 创建子类

JDK 动态代理

  1. 使用 JDK 动态代理，使用 proxy 类里的方法

    调用 newProxyInstance 方法

    返回值 参数
    static Object newProxyInstance(ClassLoader loader, Class<?>[] interfaces, InvocationHandler h)

    Param1: 类加载器

    Param2: 增强方法所在的类，这个类实现的接口，支持写多个接口

    Param3: 实现这个接口 InvocationHandler，创建代理对象，写增强的方法

    // UserDao.java
public interface UserDao {
    
    public int add(int a, int b);

    public String update(String id);

}
    // UserDaoImpl.java
public class UserDaoImpl implements UserDao{

    @Override
    public int add(int a, int b) {
        return a + b;
    }

    @Override
    public String update(String id) {
        return id;
    }
}
    // JDKProxy.java
public class JDKProxy {

    public static void main(String[] args) {
        Class[] interfaces = {UserDao.class};
        UserDaoImpl userDao = new UserDaoImpl();
        UserDao dao = (UserDao) Proxy.newProxyInstance(JDKProxy.class.getClassLoader(), interfaces, new UserDaoProxy(userDao));
        int add = dao.add(1, 2);
        String update = dao.update("Hello world");
    }
}

class UserDaoProxy implements InvocationHandler {

    // 传递要创建代理对象的类
    // 有参构造
    private Object obj;
    public UserDaoProxy(Object obj) {
        this.obj = obj;
    }

    // 增强的逻辑
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {

        // 要增强的方法之前
        System.out.println("Before..." + method.getName() + " : Params ..." + Arrays.toString(args));

        // 执行要增强的方法
        Object res = method.invoke(obj, args);

        // 要增强的方法之后
        System.out.println("After..." + obj);

        return res;
    }
}

AOP 术语

  • 连接点

    类中可以被增强的方法

  • 切入点

    实际被真正增强的方法

  • 通知（增强）

    实际增强的逻辑部分

    • 前置通知 @Before
    • 后置通知 @Afterreturning
    • 环绕通知 @Around
    • 异常通知 @AfterThrowing
    • 最终通知 @After

  • 切面

    是一个动作

    把通知应用到切入点的过程

AOP 操作

  1. Spring 框架一般都是基于 AspectJ 实现 AOP 操作

    AspectJ 不是 Spring 组成部分，独立 AOP 框架，一般把 AspectJ 和 Spring 框架一起使用，进行 AOP 操作

  2. 基于 AspectJ 实现 AOP 操作

    • 基于 xml
    • 基于注解（常用）

  3. 步骤

  4. 切入点表达式

    知道对类里哪个方法进行增强

    语法结构：execution([权限修饰符][返回类型][类全路径][方法名称]([参数列表]))

    • eg: 对 com.company.luoxin.user 类里的 add 方法进行增强

      execution(* com.company.luoxin.user.add(..))

      * 是通配符，也可以用在方法名称、类路径中

  5. 相同切入点抽取

    在方法前加入以下注解

    @Pointcut(value = "execution(* com.luoxin.User.add(..))")
public void enhance(){}

    然后再通知前加入以下注解（以 @Before 为例）

    @Before(value="enhance()")

  6. 有多个增强类对同一个方法进行增强时，可以设置增强类优先级

    在增强类上添加注解 @Order(2) ，注解中 2 可以替换成其他数字，值越小优先级越高

综上：

个人结合 《Spring 实战（第四版）》写出了如下测试代码

首先创建一个 Music 类

// Music.java
@Component
public class Music {

    public int play(int order) {
        System.out.println("Play No." + order + " music");
        return order;
    }
}

在正常调用 Music.play 方法时，只会输出一行信息，并返回传入的参数

现在问题来了，如何在不改变原方法的情况下，为 Music.play 方法的调用计数？

具体操作如下：

创建 MusicCounter 类

// MusicCounter.java
@Aspect
@Component
public class MusicCounter {
    // 记录第 order 首曲子被演唱的次数
    private Map<Integer, Integer> mp = new HashMap<>();

    // order 传参
    // 注意使用了 @Pointcut
    @Pointcut(value = "execution(* com.luoxin.spring.Music.play(..)) && args(order)")
    public void playEnhance(int order) {
    }

    // 注意额外的代码
    @Around(value = "playEnhance(order)")
    public int playAround(ProceedingJoinPoint jp, int order) throws Throwable {
        System.out.println("Let's begin!");
        int initialReturn = (int) jp.proceed();
        System.out.println("The end!");
        int currentTimes = getTimes(order);
        mp.put(order, currentTimes + 1);
        return initialReturn * 2;
    }

    public int getTimes(int order) {
        return mp.containsKey(order) ? mp.get(order) : 0;
    }
}

创建配置文件

// AppConfig.java
@Configuration
@EnableAspectJAutoProxy
public class AppConfig {

    @Bean
    public Music setMu() {
        return new Music();
    }

    @Bean
    public MusicCounter setMc() {
        return new MusicCounter();
    }
}

其实通知的过程已经结束了，以下是测试类

// MusicTest.java
@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(classes = AppConfig.class)
public class MusicTest {
    @Autowired
    Music music;

    @Autowired
    MusicCounter musicCounter;

    @Test
    public void testMusic() {

        music.play(4);

        for (int i = 0; i < 5; ++i) {

}

Question:

博主在自己演示的时候，曾经把 MusicCounter 类的 HashMap 属性设为了 static，而且为 AppConfig 添加了 @ComponentScan 注解

之后发现输出中，最后通知都输出两次，而且计数的时候，都记录了双倍：0 2 8 4 2，但 Music.play 方法的输出只有一次。

起初猜测原因在于，@ComponentScan 已为 Music 类和 MusicCounter 类创建了对象，然后 AppConfig 类中的 @Bean 注解又再次创建，致使在 MusicTest 类在运行的时候，两个对象中都运行了一次，即一个方法运行了两次。

但是又很奇怪， Music.play 方法的输出只有一次，所以上述猜测应该不成立

详细解释，还得等以后再来解答

AOP 为类添加新方法

前文都是对方法的增强，但是 AOP 也可以为类增加新的方法

比如接着上文，Music 类中，仅有 play 方法，现在我们希望增加一个 stop 方法

操作如下：

首先定义一个接口：

// Stop.java
@Component
public interface Stop {
    public void musicStop();
}

接着为这个接口添加一个实现类

// MusicStop.java
@Component
public class MusicStop implements Stop{
    @Override
    public void musicStop() {
        System.out.println("Music stop...");
    }
}

上面都是常规操作，现在注意以下内容：

编写切面

// NewMethod.java
@Aspect
@Component
public class NewMethod {
    @DeclareParents(value = "com.luoxin.spring.Music", defaultImpl = MusicStop.class)
    public static Stop stop;
}

到此就已经结束了

添加测试的话，只需要 AppConfig 类中，添加 Bean，或者直接加上 @ComponentScan 注解（添加注解后，可以去掉里面的方法）

最后再 MusicTest 中，对 music 强制转换即可（直接在上面已有的代码中插入）

((Stop) music).musicStop();

可以得到输出（已略掉其他输出）

Music stop...

Over.

综上，简而言之，AOP 基本就是首先用 IOC 注入依赖，然后 ：

  1. 在增强方法所在类上，引入 @Aspect 注解
  2. 在增强方法上，引入 @Before 等注解，注解里面 写上execution(* com.company.luoxin.user.add(..))
  3. 配置文件上加入 @EnableAspectJAutoProxy 注解
