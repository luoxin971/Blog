发表于2021-04-15|更新于2021-04-15|FixProblem|PowerShell•java
Powershell 执行 java 命令乱码问题：下载完 OpenJDK 并配好环境变量后，在 Powershell 中，输入 1java 显示乱码 Powershell 编码是 adb UTF8 解决： 添加环境变量 12JAVA_TOOL_OPTIONS-Dfile.encoding=UTF-8 或者把语言设置为英文也行 12JAVA_TOOL_OPTIONS-Duser.language=en 以下是修复之后的情况 另外，如果将 Powershell 的编码设置为 GBK 的话，输入 java 命令后，也会显示正常，但是 java 运行程序输出的中文，又会有乱码，归根结底还是，编码不一样导致乱码。
