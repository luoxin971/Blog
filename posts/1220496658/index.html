IOC 基于 xml 管理 bean

Inversion Of Control

IOC 接口

  1. IOC 思想基于 IOC 容器完成 IOC 容器底层就是对象工厂

  2. Spring 提供 IOC 容器两种实现方式：

    • BeanFactory: IOC 容器基本实现，是 Spring 内部使用的接口，不提供开发人员使用

      加载配置文件时候不会创建对象，在获取对象（使用）才去创建对象

    • ApplicationContext: BeanFactory 接口的子接口，提供更多更强大的功能，一般由开发人员进行使用

      加载配置文件时候就会把配置文件对象进行创建

  3. ApplicationContext 接口有实现类

    FileSystemXmlApplicationContext 要写绝对路径

    ClassPathXmlApplicationContext 写相对路径

IOC 操作 Bean 管理

  1. 什么是 Bean 管理
    • Spring 创建对象
    • Spring 注入属性
  2. Bean 管理操作有两种方式
    • 基于 xml 配置文件方式实现
    • 基于注解方式实现

基于 xml 方式管理

创建对象

  • 在 Spring 配置文件中，使用 bean 标签，标签里添加对应属性，就可以实现对象创建

  • 介绍常用的属性

    id 属性：唯一标识

    class 属性：类全路径（包类路径）

  • 创建对象的时候，默认执行无参构造方法完成创建

注入属性

DI: 依赖注入，就是注入属性

DI 是 IOC 的一种实现方式

使用 set 方法注入

创建属性的 set 方法

<!-- set 方法注入属性-->
<bean id="book" class="com.company.spring5.testdemo.Book">
    <!-- 使用 property 完成属性注入
    name: 类属性名称
    value: 向属性注入的值
    -->
    <property name="bookName" value="Love And Peace"></property>
</bean>
使用有参构造进行注入

创建有参构造器

<bean id="college" class="com.company.spring5.testdemo.College">
    <constructor-arg name="address" value="Xi'an"></constructor-arg>
    <constructor-arg name="name" value="NWPU"></constructor-arg>
</bean>

个人的问题有：

Q1: 如果有多个有参构造器，该如何识别

A1: name属性值就是类属性的名称

Q2: 如果参数是一个对象，该如何传值

A2: 先创建一个 bean，然后用 ref 属性，ref 属性值即为前者的 id 值

注入其他类型属性

  • 字面量

    • null 值

      使用 null 标签

      <bean id="college" class="com.company.spring5.testdemo.College">
    <constructor-arg name="location" value="Xi'an"></constructor-arg>
    <constructor-arg name="name">
        <null />
    </constructor-arg>
</bean>
<!--注意用 property 也可以-->

    • 特殊字符

      1. 转义

        &gt; &lt;

      2. 把特殊内容写进 CDATA

        <bean id="college" class="com.company.spring5.testdemo.College">
    <constructor-arg name="location" value="Xi'an"></constructor-arg>
    <constructor-arg name="name">
        <value><![CDATA[<<Xi'an>>]]></value>
    </constructor-arg>
</bean>
<!--注意用 property 也可以-->

  • 注入属性-外部 bean

    先用 bean 创建对象，然后用 ref 引用

    <bean id="book" class="com.company.spring5.testdemo.Book">
    <property name="bookName" value="Love And Peace"></property>
</bean>
<bean id="college" class="com.company.spring5.testdemo.College">
    <constructor-arg name="location" value="Xi'an"></constructor-arg>
    <constructor-arg name="name">
            <value><![CDATA[<<Xi'an>>]]></value>
    </constructor-arg>
    <property name="book" ref="book"></property>
</bean>

  • 注入属性-内部 bean

eg: 一对多 Department - Employee 在 Employee 中表示所属的 Department

在创建 Employee 的 bean 时，嵌套一个 Department 的 bean

  • 级联赋值

    两种方法

    1. 在外部 bean 中赋值，然后 ref 引用

    2. 也可以在外部创建 bean，然后用

    <property name="department.name" value="xxx"></property>
<!-- 注意 name 的值-->

  • 注入集合属性

    类属性定义如下

    private String[] coursers;
private List<String> list;
private Map<String, String> map;
private Set<String> set;
// 注意还要有相关属性的 set 方法

    xml 如下

    <bean id="student" class="com.company.spring5.collectiontype.Student">
    <property name="coursers">
        <array>
            <value>java</value>
            <value>sql</value>
        </array>
    </property>
    <property name="list">
        <list>
            <value>Jack</value>
            <value>Mary</value>
        </list>
    </property>
    <property name="map">
        <map>
            <entry key="java" value="spring"></entry>
            <entry key="cpp" value="stl"></entry>
        </map>
    </property>
    <property name="set">
        <set>
            <value>Mysql</value>
            <value>Nosql</value>
        </set>
    </property>
</bean>

    Q: 枚举的方式始终有点笨拙，如何实现类似 for 循环的遍历

    A: 暂无，或许问题也有问题？

    在集合中引用对象：

    <property name="courseList">
	<list>
        <ref bean="course1"></ref>
        <ref bean="course2"></ref>
    </list>
</property>

    将集合提取：

    1. 在 xml 配置文件中，引入 util

    2. 提取 list 集合属性注入

    <util:list id="bookList">
	<value>yyy</value>
    <value>xxx</value>
</util:list>
    <!-- 中间也可以用 ref 替换 value 来引用一个对象-->
    1. 注入使用
    <property name="list" ref="boolList"></property>

Spring 有两种 bean，一种普通 bean，另一种工厂 bean (FactoryBean)

  • 普通 bean: 在配置文件中定义 bean 类型就是返回类型（如上面所创建的 bean）
  • 工厂 bean: 在配置文件中定义 bean 类型可以和返回类型不一样

bean 作用域

在 Spring 中，设置创建 bean 实例默认是单例

Q: 如何设置是多实例还是单实例

A: scope 属性，scope 属性默认属性值是 “singleton”，可以设置为 “prototype” 来表示多实例对象

​ singleton 与 prototype 的区别：

​ singleton 在加载 Sping 配置文件的时候就会创建单实例对象

​ prototype 则是在调用 getBean 方法时创建多实例对象

bean 生命周期

  1. 通过构造器创建 bean 实例（无参构造）
  2. 为 bean 的属性设置值和对其他 bean 引用（调用 set 方法）
  3. 调用 bean 的初始化的方法（需要进行配置初始化的方法）
  4. bean 可以使用了（对象获取到了）
  5. 当容器关闭时，调用 bean 的销毁的方法（需要进行配置销毁的方法）
<bean id="course" class="com.company.spring5.collectiontype.Course" init-method="init" destroy-method="destroy">
    <property name="courseName" value="cpp"></property>
</bean>

bean 的后置处理器

  1. 通过构造器创建 bean 实例（无参构造）
  2. 为 bean 的属性设置值和对其他 bean 引用（调用 set 方法）
  3. 把 bean 实例传递 bean 前置处理器的方法
  4. 调用 bean 的初始化的方法（需要进行配置初始化的方法）
  5. 把 bean 实例传递 bean 后置处理器的方法
  6. bean 可以使用了（对象获取到了）
  7. 当容器关闭时，调用 bean 的销毁的方法（需要进行配置销毁的方法）
<bean id="myBeanPost" class="com.company.spring5.testdemo.MyBeanPost"></bean>
<!-- MyBeanPost 是 BeanPostProcessor 的实现类 -->

xml 自动装配

自动装配：根据指定装配规则（属性名称或者属性类型），Spring 自动将匹配的属性值进行注入

一般很少用

<bean id="employee" class="com.company.spring5.autowire.Employee" autowire="byName">
    <!-- autowire 常用有两个值 1. byName  2. byType
        byName 根据属性名称注入，注入值 bean 的 id 值和类属性名称一样
        byType 根据属性类型注入，但是注意不能有多个相同类型的对象
     -->
</bean>
<bean id="department" class="com.company.spring5.autowire.Department"></bean>

操作外部属性文件

  1. 引入 context 标签

    <context:property-placeholder location="classpath:jdbc.properties"/>
<!-- 注意要在 xml 文件上方导入 context 这个标签 -->

  2. 使用 ${}

    <bean id="dataSouce" class="com.aaa.bbb">
	<property name="url" value="${xxx.url}"></property>
</bean>

个人总结，xml 配置，无非就是

  • 用 <bean></bean> 标签，来创建对象，其中有 id, class 等属性，

    scope 可以用来区别单例还是多例，

    init-method, destory-method 用来传递后置处理器，

    autowire 用来自动装配

  • 用 <property></property> 标签来注入属性，其中有 name, value 等属性，

    ref 可用来注入外部对象/属性，

    对于集合类属性，可以用 array, list, set 等标签
