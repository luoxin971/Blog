Git 进阶使用

工具

该篇文章是 Git 的一些进阶使用，更为常见的配置使用可查阅 Git 命令基本使用

虽然题目为进阶使用，但其实以下内容掌握之后，才能真正在实际使用中游刃有余，因为很难预料工作中会碰到什么问题。
文章主要分为以下几个内容：分支管理 · commit 相关 · 撤销更改 · 版本回退 · 查看更改 · 其他

其中分支管理，撤销更改尤为重要

分支管理

创建分支

git branch branch_name [other_branch_name | commit]
git switch -c branch_name [other_branch_name | commit]
git checkout -b branch_name [other_branch_name | commit]

注意：

第一行只是创建一个分支，但后两个命令则是创建并切换到这个分支

后面可以接其他分支名或者提交

查看分支

git branch 
git branch -a
# 加上 -a 后查看所有分支

删除分支

git branch -d [branch_name]
git branch -D [branch_name]
# 使用 -D 强制删除
git push origin --delete [branch_name]
# 删除远程分支

更改分支名

git branch -m [branch_name] [new_branch_name]

切换分支

git switch [branch_name]
git checkout [branch_name]
# 以上两个皆可

合并分支

git merge

合并分支

git merge dev

git rebase

用于合并分支，同时讲提交变为线性

git checkout [branch_1]
git rebase [branch_2]

git rebase -i

交互性

git checkout [branch_1]
git rebase -i HEAD~5
# 然后可以在文件中编辑要合并的提交，同时还可以编辑提交的先后顺序

其他相关

git branch -f

强制将某个分支放到某个提交上

git branch -f [branch_2] [commit_2]

git cherry-pick

合并提交

git cherry-pick [commit_1] [commit_2] ...
# 合并多个提交

commit相关

HEAD 指针

~, ^

~ 后可以接数字，表示 HEAD 前第几个提交

如：

*   46dc76c (HEAD -> master) Merge branch 'b2'
|\  
| * 4bb98ef (b2) master sixth
* | 966825b master nine after change
* | 2039566 master eight
* | 35a14dd (b3) master seven
* | 1f87053 (b1) master sixth
* | 8906bd9 master fifth after change
|/  
* e2e358e b1 forth
* 02f6154 b1 third
* be7efa6 master first

如上所示

当前 HEAD 在 46dc76c 这个提交，HEAD~3 就表示 35a14dd 这个提交, HEAD^ 表示 966825b 这个提交，而 HEAD^2 表示 4bb98ef 这个提交。

注意：HEAD^ 表示上一个提交，但可见在合并分支后， 46dc76c 这个提交，后面有两个子提交，此时 HEAD^ 知道垂直的那个子提交，即 966825b, 而 HEAD^2 表示从别的分支合并来的提交，即 4bb98ef

另外 ^, ~ 可以混用，如 HEAD^2~2 即可表示 02f6154 这个提交

Ps: 如果执行 git checkout HEAD^2~2 之后，提示 HEAD detached at [commit_id], 可以通过 git switch master 来将 HEAD 重新指向 master 分支

git commit --amend

修改提交的注释

git commit --amend
# 修改最近一次提交的注释

撤销更改

# 撤销提交到暂存区
git restore --staged [file_name]
# 撤销工作区的更改
git restore [file_name]
# 这两条命令在执行 git status 的时候都会自动提示的

版本回退

git reset

相当于改写了历史，对远程分支无效（暂未验证 git push -f 是否也无效）

git reset --<hard | mixed | soft> [commit_id]
# soft 表示将这次提交之后的所有更改放入暂存区
# mixed 表示放入工作区
# hard 表示放弃所有提交，即直接删除了之间的所有提交

git revert

新建一个提交，用以撤销之前的提交的修改

例如：

* 636729f (HEAD -> bt4) bt4 nine change 4
* d23a742 bt4 nine change 3

对于上述提交历史，

执行

git revert HEAD^

后，结果为

* f03f4e4 (HEAD -> bt4) Revert "bt4 nine change 3"
* 636729f bt4 nine change 4
* d23a742 bt4 nine change 3

撤销版本回退

对于 git reset 误删的提交，可以通过 git reflog 命令查看历史，然后根据提交号，进行 git reset 即可

例如：

对于如下 commit log

* b1ec67d (HEAD -> master) master change restfile 2
* a3bbf0b master change resetfile 1
* 653c57e master add resetfile

执行

git reset HEAD^^

后得到如下 commit log

* 653c57e (HEAD -> master) master add resetfile

如果此时，要再次恢复到 b1ec67d 这个提交

则

git reflog -n 5
# 此处 -n 5 只是限制输出 5 行到作用

得到如下输出

653c57e (HEAD -> master) HEAD@{0}: reset: moving to HEAD^^
b1ec67d HEAD@{1}: commit: master change restfile 2
a3bbf0b HEAD@{2}: commit: master change resetfile 1
653c57e (HEAD -> master) HEAD@{3}: commit: master add resetfile
46dc76c (bt5) HEAD@{4}: checkout: moving from bt4 to master

可以看到曾经提交的 commit id

此时执行

git reset --hard b1ec
# 此处的 --hard 只是个人推荐吧，因为用 --soft或者 --mixed又会在工作区或者暂存区有更改，比较麻烦

即可恢复。

查看更改

本来是不打算写这部分内容的，因为 vscode 完美的直观实现了这个，但是近期在新电脑上来不及下载 vscode，所以又得用到这个。

基本上就是记住，什么都不加就是比较工作区，加了 --cached or --staged 就是比较暂存区，指定 commit-id 则不言而喻。

另外就是要学会看懂输出结果，注意以哪个文件为基准进行比较的。注意分辨！！

  • 比较工作区与暂存区

    git diff

    以暂存区为基准，不加任何参数

  • 比较暂存区与最新本地仓库

    git diff --cached [<path>...]

    以本地仓库为基准

    […] 表示文件路径，直接 copy 的，懒得替换成 [filename] 了，下同

  • 比较工作区与最新本地版本库

    git diff HEAD [<path>...]

  • 比较工作区与指定 commit_id 的差异

    git diff [commit_id] [<path>...]

    以指定 commit 为基准

  • 比较暂存区与指定 commit-id 的差异

    git diff --cached [commit_id] [<path>...]

    以指定 commit 为基准

  • 比较两个 commit-id 的差异

    git diff [commit_id] [commit_id] [<path>...]

    以前一个 commit 为基准

如执行

echo abc > seven
git diff

得到如下输出

The file will have its original line endings in your working directory

+++ b/seven
@@ -0,0 +1 @@
+abc

接着上面的更改执行

git add .
git diff --staged
# --staged 与 --cached 效果一样

得到如下输出

diff --git a/seven b/seven
index e69de29..8baef1b 100644
--- a/seven
+++ b/seven
@@ -0,0 +1 @@
+abc

而此时如果执行

git diff

则没有输出

接着执行

echo bcd > seven
git diff HEAD
git diff 
git diff --staged

其他

git show [commit_id]:[filename] 用于查看某次提交中某个文件的内容

git show a3bb:resetfile
# 输出 Hello world
